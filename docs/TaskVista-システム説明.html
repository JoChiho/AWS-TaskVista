1システム概要

TaskVista は、5〜15 名規模のチーム向けタスク管理システムです。 プロジェクト単位でメンバーを共有し、かんばん・テーブル・ダッシュボードで進捗を可視化します。 フロントエンドは SPA、バックエンドは AWS Serverless で構成します。

フロントエンド frontend/

Vue 3 + TypeScript + Vuetify 3 + Pinia。S3 + CloudFront で配信。 認証は Cognito Hosted UI（OAuth）。

バックエンド backend/

API Gateway (HTTP API) + Lambda (Node.js 20 / TypeScript) + DynamoDB + S3。 ドメイン別に Lambda を分割（projects / tasks / comments / attachments / dashboard）。

2全体構成図

ブラウザ（SPA）から AWS バックエンドまでの全体像です。 認証は Cognito、API は JWT 付きで API Gateway → Lambda → DynamoDB / S3 に到達します。 図中の番号は典型的な処理の一例であり、データの向きを一方向に固定するものではありません。

① SPA 取得（CloudFront / S3 frontend）　 ② Cognito ログイン（JWT は Hosted UI から SPA へ返却）　 ④⑤ API Gateway 経由で Lambda を invoke　 ⑥ DynamoDB へ業務データ　 ⑦⑧ 添付は URL 発行後、ブラウザと S3 が直接通信（破線）

アイコン出典: AWS Architecture Icons （公式アセットパッケージ）。

3認証フロー

認証は Cognito User Pool + Hosted UI（OAuth）。 コールバックで ID Token を受け取り、以降の API 呼び出しに付与します。

表示名: DynamoDB TaskVista-Users に姓（familyName）・名（givenName）を保存。 フルネームは displayName = "姓 名"。アバター表示は姓のみ。

4リクエストフロー

タスク更新を例にした、フロントから永続化までの流れです。

5添付ファイルフロー

Lambda のペイロード上限を避けるため、ファイル本体はブラウザから S3 へ直接送受信します。

9ローカル開発

Lambda 相当の処理だけローカルで実行し、DynamoDB / Cognito / S3 はクラウドを共有します。 

# 端末1
cd backend && npm run build

# 端末2
npm run local:api

# 端末3
cd frontend && npm run dev
# VITE_API_BASE_URL=http://127.0.0.1:3001
Cognito アプリクライアントに http://localhost:5173/callback と http://localhost:5173/login を登録してください。

10デプロイ

本番反映はルートの scripts/deploy.mjs を使用します。

npm run deploy              # 全体
npm run deploy:backend      # Lambda 等
npm run deploy:frontend     # S3 + CloudFront 無効化

設定は deploy.config.json。 template.yaml は主に SAM ローカル向けです。

11セキュリティ

  • API は JWT 必須（ローカル開発時のみモックヘッダ可）
  • プロジェクト権限は createdBy / memberIds / 招待メールで判定
  • メンバー追加は原則オーナーのみ
  • 添付は時間制限付き Presigned URL
  • Cognito Admin API はフロントから直接呼ばない
  • 相関 ID（X-Correlation-Id）でリクエスト追跡
