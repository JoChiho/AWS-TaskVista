1システム概要

TaskVista は、5〜15 名規模のチーム向けタスク管理システムです。 プロジェクト単位でメンバーを共有し、かんばん・テーブル・タイムライン・ダッシュボードで進捗と日程を可視化します。 フロントエンドは SPA、バックエンドは AWS Serverless で構成します。

フロントエンド frontend/

Vue 3 + TypeScript + Vuetify 3 + Pinia。S3 + CloudFront で配信。 認証は Cognito Hosted UI（OAuth）。

バックエンド backend/

API Gateway (HTTP API) + Lambda (Node.js 20 / TypeScript) + DynamoDB + S3。 ドメイン別に Lambda を分割（projects / tasks / comments / attachments / dashboard）。

主な画面

画面内容
ダッシュボード プロジェクト概要、自分の担当タスク、 レビュー待ちのタスク（自分がレビュアー）
プロジェクト一覧作成・編集・詳細・メンバー管理
かんばん ステータス列のドラッグ＆ドロップ。 要レビューフィルタで対象を強調
テーブルソート・フィルタ・一覧（開始日・工数・レビュアー列含む）
タイムライン 開始日 + 予定工数（人日）によるガント風カレンダー。 未設定タスクのドラッグで開始日設定、全体/自分切替
タスク詳細 要件・説明・進捗・開始日・工数・締切・複数担当・ レビュアー・コメント・添付

2全体構成図

ブラウザ（SPA）から AWS バックエンドまでの全体像です。 認証は Cognito、API は JWT 付きで API Gateway → Lambda → DynamoDB / S3 に到達します。 図中の番号は典型的な処理の一例であり、データの向きを一方向に固定するものではありません。

① SPA 取得（CloudFront / S3 frontend）　 ② Cognito ログイン（JWT は Hosted UI から SPA へ返却）　 ④⑤ API Gateway 経由で Lambda を invoke　 ⑥ DynamoDB へ業務データ　 ⑦⑧ 添付は URL 発行後、ブラウザと S3 が直接通信（破線）

アイコン出典: AWS Architecture Icons （公式アセットパッケージ）。

3認証フロー

認証は Cognito User Pool + Hosted UI（OAuth）。 コールバックで ID Token を受け取り、以降の API 呼び出しに付与します。

表示名: DynamoDB TaskVista-Users に姓（familyName）・名（givenName）を保存。 フルネームは displayName = "姓 名"。アバター表示は姓のみ。

4リクエストフロー

タスク更新を例にした、フロントから永続化までの流れです。

5添付ファイルフロー

Lambda のペイロード上限を避けるため、ファイル本体はブラウザから S3 へ直接送受信します。

8主要機能

プロジェクト

作成・編集・論理削除、ステータス、メンバー招待（オーナー権限）、 ダッシュボード概要（人数・最終更新・タスク集計）。

タスク（基本）

かんばん / テーブル / タイムライン、複数担当、進捗連動、 予定工数（人日）、開始日・締切日、要件・説明、コメント、添付。

タイムライン（ガント風カレンダー）

開始日を基準に工数分のバーを表示。全体 / 自分のタスク切替。 開始日未設定（保留以外）は日付へドラッグして設定。 ドラッグ中は他タスクを隠し、日付カレンダーのみ表示。

レビュアー・レビュー

レビュー待ちでプロジェクトメンバーをレビュアーに複数指定。 完了後も保持表示。ダッシュボードに「レビュー待ちのタスク」欄、 かんばんに「要レビュー」フィルタとカード強調。

画面ビュー切替（プロジェクト内）

ビューパス用途
かんばん/projects/:id/boardステータス移動・要レビュー強調
テーブル/projects/:id/table一覧・ソート・フィルタ
タイムライン/projects/:id/timeline日程バー・ドラッグで開始日設定

ダッシュボード構成

  1. プロジェクト概要 … ステータス別タスク数・メンバー数・最終更新
  2. 自分の担当タスク … 未完了の担当タスク（締切昇順）
  3. レビュー待ちのタスク … レビュー待ちかつ自分がレビュアー（担当欄の下）

9ローカル開発

Lambda 相当の処理だけローカルで実行し、DynamoDB / Cognito / S3 はクラウドを共有します。 

# 端末1
cd backend && npm run build

# 端末2
npm run local:api

# 端末3
cd frontend && npm run dev
# VITE_API_BASE_URL=http://127.0.0.1:3001
Cognito アプリクライアントに http://localhost:5173/callback と http://localhost:5173/login を登録してください。

10デプロイ

本番反映はルートの scripts/deploy.mjs を使用します。

npm run deploy              # 全体（Lambda + API ルート同期 + フロント）
npm run deploy:backend      # Lambda 更新 + 不足 API ルート同期
npm run deploy:frontend     # S3 sync + CloudFront 無効化
npm run deploy:routes       # API Gateway ルート同期のみ

設定は deploy.config.json。 template.yaml は主に SAM ローカル向けです。 インフラを CDK で管理する場合は infra/（Auth / Storage / Backend / Frontend）。

新 API パスを追加したあと: Lambda だけ更新しても API Gateway にルートが無いと 404 になります。 deploy:backend または deploy:routes で API_ROUTES（scripts/lib/config.mjs）と差分同期してください。

11セキュリティ

  • API は JWT 必須（ローカル開発時のみモックヘッダ可）
  • プロジェクト権限は createdBy / memberIds / 招待メールで判定
  • メンバー追加は原則オーナーのみ
  • 添付は時間制限付き Presigned URL
  • Cognito Admin API はフロントから直接呼ばない
  • 相関 ID（X-Correlation-Id）でリクエスト追跡
  • 表示名は Users テーブルに保持（Cognito クレームの生 username に依存しない）
